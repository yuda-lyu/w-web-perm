# 應用啟動與登入流程

## 觸發

瀏覽器使用者開啟後台網址，Vue 根元件 `App.vue` 於 `beforeMount` 啟動：先 `setVo` 綁定廣播用 vo、`setLang` 依 `window.___pmwperm___.language` 初始化語系，再以 `wui('wperm', ...)`（`w-ui-loginout`）建立登入輔助物件並呼叫其 `login()`，由 token（網址 query 或 `localStorage` 的 `wperm:userToken`）取得使用者資訊。

登入結果分兩路：成功則回呼 `loginSuccess`，將 `connState` 設為 `csLogin`、寫入 `userToken` 與 `userSelf`；失敗則回呼 `loginError`，將 `connState` 設為 `csErrLogin`、清空 `userToken`，並於正式環境重導至 `urlRedirect`（開發環境延遲 60 秒後重導）。

畫面在 `ready` 為真之前一律由 `LayoutState.vue` 依 `connState` 顯示連線狀態畫面（`csIng` / `csLogin` / `csLogout` / `csErrConn` / `csErrLogin` 五種）。`ready`（`App.vue:128`）需同時滿足 `connState==='csLogin'` 且 `webInfor` 為非空物件才為真，為真後切換為 `Layout.vue` 殼層，內含 `LayoutContent.vue` 五頁籤（`mmTargets` / `mmPemis` / `mmGrups` / `mmUsers` / `mmStaInfor`）。`webInfor` 由 `main.js` 的 `WServHapiClient` 取得 `$fapi` 後呼叫 `getWebInfor` 取回並 commit、同時 commit `syncState=true`。

## 重要流程

本流程五案皆於 e2e-init 以「後端注入語系」驗證（後端以指定 `language` 啟動 → dist 注入 `window.___pmwperm___.language`，連線前狀態畫面走 fallback、登入後殼層走 `webInfor.language`，皆非前端 `setLang` 切換）；案序依使用者觀察之時序：連線中 → 已登入過場 → 登入成功（五頁籤）→（失敗分支）拒絕登入 / 無法連線。

## spec 規則摘要

- **啟動序**：`App.vue` `beforeMount` 依序 `setVo` → `setLang` → 建立 `wui('wperm')` → `login()`；`webInfor` / `syncState` 由 `main.js` 的 `WServHapiClient` 經 `getWebInfor` 另路回填。
- **進入後台條件**：`ready`（`App.vue:128`）須同時 `connState==='csLogin'` 且 `webInfor` 為非空物件；任一未滿足皆停留在 `LayoutState` 連線狀態畫面。
- **狀態畫面**：`LayoutState` 依 `connState` 五分支渲染（`csIng` / `csLogin` / `csLogout` / `csErrConn`，其餘落 `v-else` 的 `csErrLogin`）；連線前文字由 `mUI.mjs` 的 `kpFallback` 提供雙語預設，後端語系載入後改取 `kpText`。
- **登入分流**：`login()` 成功 → `loginSuccess` 設 `csLogin` + 寫 `userToken` / `userSelf`；失敗 → `loginError` 設 `csErrLogin` + 清 `userToken`，並重導 `urlRedirect`（開發環境延遲 60 秒、正式環境即時）。
