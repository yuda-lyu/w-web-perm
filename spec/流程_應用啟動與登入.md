# 應用啟動與登入流程

## 觸發

瀏覽器使用者開啟後台網址，Vue 根元件 `App.vue` 於 `beforeMount` 啟動：先 `setVo` 綁定廣播用 vo、`setLang` 依 `window.___pmwperm___.language` 初始化語系，再以 `wui('wperm', ...)`（`w-ui-loginout`）建立登入輔助物件並呼叫其 `login()`，由 token（網址 query 或 `localStorage` 的 `wperm:userToken`）取得使用者資訊。

登入結果分兩路：成功則回呼 `loginSuccess`，將 `connState` 設為 `csLogin`、寫入 `userToken` 與 `userSelf`；失敗則回呼 `loginError`，將 `connState` 設為 `csErrLogin`、清空 `userToken`，並於正式環境重導至 `urlRedirect`（開發環境延遲 60 秒後重導）。

畫面在 `ready` 為真之前一律由 `LayoutState.vue` 依 `connState` 顯示連線狀態畫面（`csIng` / `csLogin` / `csLogout` / `csErrConn` / `csErrLogin` 五種）。`ready`（`App.vue:128`）需同時滿足 `connState==='csLogin'` 且 `webInfor` 為非空物件才為真，為真後切換為 `Layout.vue` 殼層，內含 `LayoutContent.vue` 四頁籤（`mmTargets` / `mmPemis` / `mmGrups` / `mmUsers`）。`webInfor` 由 `main.js` 的 `WServHapiClient` 取得 `$fapi` 後呼叫 `getWebInfor` 取回並 commit、同時 commit `syncState=true`。

## 重要流程

- **E2E-001**
  - title: 開啟後台時先顯示連線中畫面
  - description: 驗證使用者開啟後台網址、登入驗證尚未完成（`connState` 仍為初始值 `csIng`）時，`LayoutState` 以連線動畫圖示加 `csIng` 文字呈現連線中過場畫面，尚未進入後台。此為應用啟動的初始過場態，後續成功 / 失敗畫面皆由此態切換而來。
  - flow:
    - 測試資料：base seed 既有 users / webInfor，不另建特化資料；以初始 `connState='csIng'` 起始態觀察。
    - 操作：開啟後台網址，於登入驗證完成前觀察過場畫面。
    - 驗證：
      1. 語意：畫面出現 `csIng` 文字（連線中），未出現四頁籤導覽。
      2. 視覺：與 baseline `test/pics/startup/startup-eng-E2E-001-conn-ing.png` 視覺一致（pixelmatch 容差）。
    - 雙語：僅 eng 一份（此連線狀態畫面文字語系由 server 注入之 `___pmwperm___.language` 決定、非 UI 可切；雙語覆蓋見本流程「初始畫面語系(server 注入)」章節）。
    - 清理：本案唯讀過場態，不改 DB，無需清理。

- **E2E-002**
  - title: 登入成功進入後台顯示四頁籤
  - description: 驗證使用者持有效 token，`login()` 取得使用者資訊成功、`loginSuccess` 將 `connState` 設為 `csLogin` 並寫入 `userToken` / `userSelf`，且 `getWebInfor` 回填 `webInfor`、`syncState` 為真後，`ready` 轉真，畫面由 `LayoutState` 切換為 `Layout` 殼層，`LayoutContent` 顯示左側四頁籤（`mmTargets` / `mmPemis` / `mmGrups` / `mmUsers`），預設停在 `mmTargets`。此為啟動成功 golden 終態。
  - flow:
    - 測試資料：base seed users / webInfor；以有效 token 完成登入。
    - 操作：以有效 token 開啟後台網址，等待登入與首次同步完成。
    - 驗證：
      1. 語意：左側導覽出現 `mmTargets` / `mmPemis` / `mmGrups` / `mmUsers` 四頁籤文字；頂列顯示 `webName`；連線狀態過場畫面已消失。
      2. 視覺：與 baseline `test/pics/startup/startup-{eng,cht}-E2E-002-login-ok.png` 視覺一致（pixelmatch 容差）。
    - 雙語：eng / cht 各一輪。
    - 清理：本案唯讀，不改 DB，無需清理。

- **E2E-003**
  - title: 登入被拒顯示拒絕登入畫面
  - description: 驗證使用者以無效 / 過期 token（或後端驗證 `state==='error'`）開啟後台時，`login()` 走 reject、`loginError` 將 `connState` 設為 `csErrLogin` 並清空 `userToken`，`LayoutState` 以拒絕圖示加 `csErrLogin` 文字顯示拒絕登入畫面，不進入後台。屬登入失敗分支終態，與成功路徑共同覆蓋登入成敗。

    - 清理：本案被拒未寫入登入態，不改 DB，無需清理。

### 初始畫面語系(server 注入)

- **E2E-006-initlang-connecting**

## spec 規則摘要

- **啟動序**：`App.vue` `beforeMount` 依序 `setVo` → `setLang` → 建立 `wui('wperm')` → `login()`；`webInfor` / `syncState` 由 `main.js` 的 `WServHapiClient` 經 `getWebInfor` 另路回填。
- **進入後台條件**：`ready`（`App.vue:128`）須同時 `connState==='csLogin'` 且 `webInfor` 為非空物件；任一未滿足皆停留在 `LayoutState` 連線狀態畫面。
- **狀態畫面**：`LayoutState` 依 `connState` 五分支渲染（`csIng` / `csLogin` / `csLogout` / `csErrConn`，其餘落 `v-else` 的 `csErrLogin`）；連線前文字由 `mUI.mjs` 的 `kpFallback` 提供雙語預設，後端語系載入後改取 `kpText`。
- **登入分流**：`login()` 成功 → `loginSuccess` 設 `csLogin` + 寫 `userToken` / `userSelf`；失敗 → `loginError` 設 `csErrLogin` + 清 `userToken`，並重導 `urlRedirect`（開發環境延遲 60 秒、正式環境即時）。
- **i18n 重渲染**：`setLang` 更新 `lang` / `kpText` 後以 `forceUpdate` 廣播全組件重繪（`mUI.mjs:191`）；語言選單僅在 `webInfor.showLanguage==='y'` 時顯示（`Layout.vue:99-101`）。
- **四頁籤**：進入後台後 `LayoutContent` 在 `syncState` 為真時渲染 `mmTargets` / `mmPemis` / `mmGrups` / `mmUsers` 四頁籤，`menuKey` 預設 `mmTargets`，點選切換對應內容組件（`LayoutContent.vue:161,72-90`）。
