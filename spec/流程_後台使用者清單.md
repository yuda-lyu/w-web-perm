# 後台使用者清單流程

## 觸發

管理員（瀏覽器使用者，經 w-ui-loginout 登入取得有效 token、`connState='csLogin'`）進入後台後，於 `LayoutContent.vue` 左側導覽點「使用者」（`mmUsers`）進入使用者清單頁（`LayoutContentUsers.vue`）。此頁以 ag-grid（`WAggridVue`）列出 `users` 表，提供：編輯模式切換、顯示欄位挑選、新增 / 複製 / 刪除 / 儲存、`cgrups` 群組關聯入口按鈕、`isAdmin` / `isActive` 就地切換。

寫入端點 `updateUsers` 經 hapi `verifyConn` → `checkToken(token)` 守門（`server/WWebPerm.mjs:1155,1171`），**token 無效 / 過期一律拒絕連線**；是否限定管理員由部署方的 `verifyBrowserUser` 決定（開發樣本限 `isAdmin='y'`，`srv.mjs`）。

| 觸發情境 | i18n 鍵（grep `server/procLang.mjs`） | 顯示位置 | 粒度規則 |
|---|---|---|---|
| name 空 | `userNameEmpty` | cell 警告 icon + tooltip | **具體**：逐 cell 標示是哪一格出錯 |
| name 重複 | `userNameDuplicate` | cell 警告 icon + tooltip | 具體（僅前端提示，後端不擋） |
| email 空 | `userEmailEmpty` | cell 警告 icon + tooltip | 具體 |
| email 格式錯 | `userEmailError` | cell 警告 icon + tooltip | 具體 |

| Save 時有任一 name 錯誤 | `errInNames` | `$dg.showCheckYes` 持久 modal | **彙整粗粒度**：N 種 name 錯誤聚合為 1 句擋存 |
| Save 時有任一 email 錯誤 | `errInEmails` | `$dg.showCheckYes` 持久 modal | 彙整粗粒度：N 種 email 錯誤聚合為 1 句擋存 |

## spec 規則摘要

- **權限**：所有 `updateUsers` 寫入經 `verifyConn` → `checkToken` 守門（有效 token 才可寫）；是否限管理員由部署方 `verifyBrowserUser` 決定（開發樣本限 `isAdmin='y'`）。
- **粒度**：欄位錯誤走 cell 警告 icon + tooltip（具體 5 種）；Save 擋存彙整（`errInNames` / `errInEmails` 粗 2 種）與操作結果（成功 / 失敗 / 空）一律走 `$dg.showCheckYes` 持久 modal（showCheckYes 前先 `updateLoading(false)`）；非預期例外（外層 catch）保留 `$alert` toast。
- **邊界**：
  - `users` **允許 name 重複**（僅前端警告、後端不擋）；`email` 不可空 / 不可重複（前端 + 後端 `ckKey` 雙守門）。
  - CRUD 為「單頁多動作 + 批次儲存」模型：`addItem` / `copyItem` / `deleteItemsCheck` / `toggle*` / `showVeCgrupsById` 只改 `opt.rows` 並設 `isModified=true`，**不即時打 API**；一律 Save 才以 `ltdtDiffByKey` 批次 del / insert / save 送出。
  - 稽核欄位（`userId` / `time*`）前端填 `{待自動給予}` 佔位符，後端 `procOrm` 補實值。
  - `isError` / `errItemsByName` / `errItemsByEmail` 為前端 computed 驗證；name 重複只警告不擋存（僅 email 與必填擋）。
  - loading 於 `core()` 內、在 `isError` / rows 空兩同步檢查**之後**才開啟（`LayoutContentUsers.vue:1188`）；早退分支不開 loading，符合全域 §5.1 canonical（`updateLoading` 應在同步檢查後）。
- **契約**：`saveUsers` 為按鈕 handler，第一行 `msg.pm.resolve()` 釋放視覺鎖後 fire-and-forget 呼叫 `doSaveUsers`；`doSaveUsers` 內 `core() + finally`，`finally` 一處關 loading（`LayoutContentUsers.vue:1215`）；`updateUsers` 失敗由其 inline `.catch` 顯示失敗提示，並以 `okSave` 旗標 `if (!okSave) return` 短路後續。
