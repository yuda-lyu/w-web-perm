# 後台權限清單流程

## 觸發

管理員（瀏覽器使用者，經登入取得有效 token、連線通過 `verifyConn` 守門）進入後台後，於左側導覽進入權限清單頁（`LayoutContentPemis.vue`）。此頁以 ag-grid（`WAggridVue`）列出 `pemis`（權限設定）表，提供：編輯模式切換、顯示欄位挑選、新增 / 複製 / 刪除 / 儲存，以及 `belongGrups` 欄（檢視該權限所屬權限群組）與 `crules` 欄（編輯該權限對管控對象的 y/n 規則）兩個關聯入口按鈕。

寫入端點 `updatePemis` 經 hapi `verifyConn` → `checkToken(token)` 守門（`server/WWebPerm.mjs:1155,1171`），**token 無效 / 過期一律拒絕連線**；是否限定管理員由部署方的 `verifyBrowserUser` 決定。

## spec 規則摘要

- **權限**：所有 `updatePemis` 寫入經 `verifyConn` → `checkToken` 守門（有效 token 才可寫）；是否限管理員由部署方 `verifyBrowserUser` 決定。
- **粒度**：欄位錯誤走 cell 警告 icon + tooltip（具體 2 種：name 空 / name 重複）；Save 擋存彙整走 `showCheckYes` 持久 modal（粗 1 種 `errInNames`）；操作結果（成功 / 失敗 / rows 空）一律 `showCheckYes` 持久 modal（成功 `pemiSavePemisSuccess` / 失敗 `pemiSavePemisFail` / 空 `pemiAddEmpty`）；唯外層 `.catch` 之非預期例外（`anUnexpectedErrorOccurred`）走 `$alert` toast 低調呈現。
- **邊界**：
  - `pemis` **不允許 name 重複**（前端 `errItemsByName` 警告、後端 `ckKey('name')` 雙守門，`WWebPerm.mjs:406`）；後端 diff key 亦為 `name`（`WWebPerm.mjs:462`）。
  - CRUD 為「單頁多動作 + 批次儲存」模型：`addItem` / `copyItem` / `deleteItemsCheck` / `showVeCrulesById` 只改 `opt.rows` 並設 `isModified=true`，**不即時打 API**；一律 Save 才以 `ltdtDiffByKey` 批次 del / insert / save 送出。
  - 兩個關聯入口：`crules` 按鈕開 `VeCrules`（resolve 回填 `crules` 字串、可編輯）；`belongGrups` 按鈕開 `VePemiBlngGrups`（resolve 不回填，屬檢視性入口）。
  - `belongGrups` 為前端 computed 衍生顯示欄，非 `pemis` schema 持久欄位，不寫入 DB。
  - 稽核欄位（`userId` / `time*`）前端填 `{待自動給予}` 佔位符，後端 `procOrm` 補實值。
  - `isError` / `errItemsByName` 為前端 computed 驗證（`LayoutContentPemis.vue:584,544`）；只要 `errItemsByName` 非空即彙整為 `errInNames` 擋存。
- **契約**：`savePemis` 採 `core() + finally`，`finally` 一處關 loading（`LayoutContentPemis.vue:1241`）；`updatePemis` 失敗由 `errTemp` 旗標短路後續、走失敗結果 modal。loading 於 `core()` 內開啟，但在同步檢查（`isError` / rows 空）**之前**（`LayoutContentPemis.vue:1191`）；填錯而早退時會先閃一下 loading（每個 early-return 分支內各自 `updateLoading(false)` 後再 `showCheckYes`）——與全域 §5.1 canonical（`updateLoading` 應在同步檢查後）有出入，屬既有行為觀察，記錄供後續決定是否調整。
