# 群組權限關聯流程

## 觸發

管理員（瀏覽器使用者，經 w-ui-loginout 登入取得有效 token、後端 `verifyConn` 守門）進入後台後，可由兩個入口進入「群組與權限的關聯」編輯。兩入口操作的是同一份資料 `grups.cpemis`（一個 JSON5 字串，結構為 `{ 權限名: { mode: 'OR'|'AND', isActive: 'y'|'n' } }`），差別只在切入角度：

- **群組清單頁入口**：於 `LayoutContentGrups.vue` 列出 `grups` 表，`cpemis` 欄渲染為一顆按鈕（顯示已啟用權限數量）；點該按鈕呼叫 `showVeCpemisById(id)`（`LayoutContentGrups.vue:225,1126`），開啟群組權限關聯對話框 `VeCpemis`。此對話框以「該群組擁有哪些權限」為視角，逐列權限可勾選啟用（`enable`）、選 OR / AND 合併模式（`mode`）。

- **權限清單頁入口**：於 `LayoutContentPemis.vue` 列出 `pemis` 表，`belongGrups` 欄渲染為一顆按鈕（顯示隸屬群組數量）；點該按鈕呼叫 `showVePemiBlngGrupsById(id)`（`LayoutContentPemis.vue:220,1086`），開啟權限隸屬群組對話框 `VePemiBlngGrups`。此對話框以「該權限被哪些群組使用」為視角，逐列群組可勾選是否把此權限掛入該群組（`enable`）、選 OR / AND 模式（`mode`）。

兩入口的對話框是否可編輯，皆由各自清單頁的編輯模式 `isEditable` 決定並隨 `show()` 傳入（`LayoutContentGrups.vue:1158`、`LayoutContentPemis.vue:1119`）：`isEditable=true` 顯示「群組權限編輯」/「權限隸屬群組編輯」標題與功能列、可勾選與存檔；`isEditable=false` 顯示「（檢視）」標題（`grupEditCpemisForDisplay` / `pemiBlngEditGrupsForDisplay`），下拉與 checkbox 一律 `disabled`、隱藏功能列與 Save 按鈕，僅供唯讀檢視。

兩入口的存檔落點不同（皆寫入同一張 `grups` 表）：

- **VeCpemis（群組頁入口）**：對話框 Save 不直接打 API，而是把編輯後的 `cpemis` 字串 resolve 回群組頁；群組頁於 `showVeCpemisById.then` 寫回該列 `opt.rows[kr].cpemis` 並設 `isModified=true`（`LayoutContentGrups.vue:1162-1173`），真正寫入 DB 延到管理員按群組頁工具列的 `saveGrups` 按鈕 → `updateGrups`。
- **VePemiBlngGrups（權限頁入口）**：對話框 Save 自帶存檔，於 `clickSave → core()` 內直接呼叫 `saveGrups(grupsNew)` → `$fapi.updateGrups(rows)` 即時寫入 DB（`VePemiBlngGrups.vue:1072,973`），存檔成功即關閉對話框。

寫入端點 `updateGrups` 經 hapi `verifyConn` → `checkToken(token)` 守門（`WWebPerm.mjs:1159,1171`），**token 無效 / 過期一律拒絕連線**。

## spec 規則摘要

- **權限**：兩入口存檔皆經 `verifyConn → checkToken` 守門（有效 token 才可寫）；後端 `ckKey('name')` 確保 `grups` name 不可空 / 不可重複。
- **同源異視角**：群組頁入口（VeCpemis）與權限頁入口（VePemiBlngGrups）編輯的是同一份 `grups.cpemis`，僅切入視角不同——前者以「群組擁有哪些權限」逐列權限，後者以「權限被哪些群組使用」逐列群組。
- **存檔落點差異**：
  - VeCpemis（群組頁入口）Save **不即時打 API**，只把 `cpemis` 字串 resolve 回群組頁該列並設 `isModified=true`；真正 DB 寫入延到群組頁工具列 `saveGrups → updateGrups`。
  - VePemiBlngGrups（權限頁入口）Save **自帶即時 DB 寫入**，於 `core() → saveGrups → $fapi.updateGrups` 直接落庫、成功後關閉對話框。
- **唯讀檢視**：`isEditable=false` 時兩對話框皆顯示檢視版標題、隱藏功能列與 Save、下拉與 checkbox 一律 disabled，僅供檢視不可改。
- **粒度**：對話框內切換防呆（無 name / 無目標）走 `$alert`；存檔結果（成功 / 失敗 / 空）一律 `$alert` toast；群組 name 錯誤 Save 擋存彙整為 `errInNames`。
- **契約**：兩入口的對話框 Save 與群組頁 `saveGrups` 皆採 `core() + .then/.catch/.finally`，`finally` 一處關 loading；`updateGrups` 失敗由 `errTemp` 旗標短路後續、走 `grupSaveGrupsFail` 失敗提示。
