# 後台標的清單流程

## 觸發

管理員（瀏覽器使用者，經 w-ui-loginout 登入取得有效 token、連線守門通過）進入後台後，於左側導覽點「管理對象」（`mmTargets`）進入標的清單頁（`LayoutContentTargets.vue`）。此頁以 ag-grid（`WAggridVue`）列出 `targets` 表，提供：編輯模式切換、顯示欄位挑選、新增 / 複製 / 刪除 / 儲存。`targets` 的主鍵 `id` 為**路徑式字串**（如「專案A/頁A/區塊A」），於編輯模式可就地編輯，並作為列首固定欄與勾選 / 拖曳欄。

寫入端點 `updateTargets` 經 hapi `verifyConn` → `checkToken(token)` 守門（`server/WWebPerm.mjs:1155,1171`），**token 無效 / 過期一律拒絕連線**；是否限定管理員由部署方的 `verifyBrowserUser` 決定。

## 重要流程

- **E2E-001**
  - title: 管理員進入標的清單頁顯示初始清單
  - description: 驗證管理員以有效 token 登入後台、於左側點「管理對象」頁籤後，`LayoutContentTargets` 以 ag-grid 渲染 `targets` 表初始清單（預設顯示 id / description 欄），呈現預設編輯模式檢視態（`webInfor.modeEditTargets='y'`，進頁即為編輯態，工具列顯示新增 / 顯示欄位按鈕、id 欄具勾選與拖曳控制）。此為本流程 golden 起點，其餘新增 / 複製 / 刪除 / 驗證失敗案例皆以此檢視態為基準。
  - flow:
    - 測試資料：base seed 既有 targets（路徑式 id，如「專案A/頁A/區塊A」一系列），不另建特化 target。
    - 操作：登入後台 → 點「Targets」/「管理對象」頁籤。
    - 驗證：
      1. 語意：表格出現 base seed 各 target 的路徑式 id 與其 description；預設編輯模式下工具列顯示新增按鈕、id 欄具勾選框。
      2. 視覺：與 baseline `test/pics/targets/targets-{eng,cht}-E2E-001-list-view.png` byte-equal。
    - 雙語：eng / cht 各一輪。
    - 清理：本案唯讀，不改 DB，無需清理。

## 欄位邊界

| 欄位 | 前端處理 | 後端處理 |
|---|---|---|
| `id` | 路徑式字串主鍵；`funNew` 以 `genIDSeq` 預給、新增 / 複製時再以 `getNameNew` 帶唯一名（`LayoutContentTargets.vue:696,762`）；編輯模式可就地編輯（`kpCellEditable`）、為列首固定欄、勾選欄與拖曳欄（`LayoutContentTargets.vue:573,580,605`）；`errItemsById` 偵測空（`targetIdEmpty`）/ 重複（`targetIdDuplicate`），於 cell 顯示橘字 + 警告 icon | `ckKey('id')` 檢查空 / 重複，違反即 `Promise.reject`（`WWebPerm.mjs:399`）；為 `ltdtDiffByKey` 的 diff key（判定 del / add / save）|
| `description` | 編輯模式可就地編輯（`kpCellEditable`，`LayoutContentTargets.vue:574`）；預設顯示欄 | 隨列 insert / save |
| `from` | 編輯模式可就地編輯（`kpCellEditable`，`LayoutContentTargets.vue:575`）；預設隱藏（不在 `tabKeysShow`）；供外部批次變更標記用 | 隨列儲存 |
| `userId` / `timeCreate` / `userIdUpdate` / `timeUpdate` | 新增 / 複製時填 `{待自動給予}` 佔位符（`targetAddIdNew`，`LayoutContentTargets.vue:697-700`）；預設隱藏 | `procOrm` 於 insert / save 時自動補實值 |
| `order` | 由清單順序決定（可拖曳改序） | `updateTabItems` 依列順序重給（`WWebPerm.mjs:360`） |

## i18n 訊息粒度規則

| 觸發情境 | i18n 鍵（grep `server/procLang.mjs`） | 顯示位置 | 粒度規則 |
|---|---|---|---|
| id 空 | `targetIdEmpty` | cell 警告 icon + tooltip | **具體**：逐 cell 標示是哪一格出錯（僅前端視覺提示，不參與 Save 擋存）|
| id 重複 | `targetIdDuplicate` | cell 警告 icon + tooltip | 具體（僅前端視覺提示，不參與 Save 擋存）|
| rows 為空 | `targetAddEmpty` | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作前置檢查 |
| 儲存成功 | `targetSaveTargetsSuccess` | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作結果 |
| 儲存失敗（後端 reject，含 id 空 / 重複 / token 失效） | `targetSaveTargetsFail`（後接錯誤字串） | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作結果：N 種後端拒因聚合為 1 句失敗提示 |
| 非預期例外（外層 catch） | `anUnexpectedErrorOccurred` | `$alert` toast | 非預期錯誤低調呈現 |

> 粒度觀察（對齊全域 §6.2）：id 錯誤在 **cell 層為具體訊息（空 / 重複 2 種）**，但此 cell 提示**不參與 Save 擋存**——`saveTargets` 的同步 `isError` 檢查引用本頁未定義的 `errItemsByName`（`LayoutContentTargets.vue:490`）故恆為空，id 空 / 重複實際由**後端 `ckKey('id')` 攔截**並以 `$dg.showCheckYes` 持久 modal 呈現 `targetSaveTargetsFail`。寫測試時，id 錯誤案例應斷言 cell tooltip 的具體鍵（`targetIdEmpty` / `targetIdDuplicate`），並斷言 Save 後 modal 出現後端拒因聚合的 `targetSaveTargetsFail`。

## 參數來源

| 概念 | 來源 | 流程影響 |
|---|---|---|
| 操作者 token | 前端 store（登入後 token，`$fapi` 帶 Bearer） | 後端 `verifyConn` → `checkToken` 守門（`WWebPerm.mjs:1155,1171`）；失效則 `updateTargets` reject 走失敗提示 |
| 操作者 userId | 後端 `getUserIdByToken` 由 token 解出（`WWebPerm.mjs:1176`） | 傳入 `procOrm` 作為稽核欄位（`userId` / `userIdUpdate`）來源 |
| `webInfor.modeEditTargets` / `showModeEditTargets` | 後端 `getWebInfor` | 決定本頁是否顯示編輯開關、預設是否進編輯模式（`LayoutContentTargets.vue:341-344`） |
| `webInfor.widthTargetId` / `widthTargetDescription` | 後端 `getWebInfor` | 決定 id / description 欄初始寬度（`LayoutContentTargets.vue:346-347,610-611`） |

## spec 規則摘要

- **權限**：所有 `updateTargets` 寫入經 `verifyConn` → `checkToken` 守門（有效 token 才可寫）；是否限管理員由部署方 `verifyBrowserUser` 決定。
- **粒度**：id 錯誤走 cell 警告 icon + tooltip（具體 2 種，僅視覺提示）；操作主要結果（成功 / 失敗 / 空）一律 `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕，對齊全域 §5.1 禁原生 alert），非預期例外走 `$alert` toast 低調呈現；id 空 / 重複的實際擋存在後端 `ckKey('id')`，前端聚合為單一 `targetSaveTargetsFail`。
- **邊界**：
  - `targets` 主鍵 `id` 為**路徑式字串**（階層以「/」分隔），**不可空 / 不可重複**（後端 `ckKey('id')` 守門，`WWebPerm.mjs:399`），且為 `ltdtDiffByKey` 的 diff key——改寫 id 等同 del 舊列 + add 新列。
  - CRUD 為「單頁多動作 + 批次儲存」模型：`addItem` / `copyItem` / `deleteItemsCheck` 及就地編輯只改 `opt.rows` 並設 `isModified=true`，**不即時打 API**；一律 Save 才以 `ltdtDiffByKey` 批次 del / insert / save 送出。
  - 稽核欄位（`userId` / `time*`）前端填 `{待自動給予}` 佔位符，後端 `procOrm` 補實值。
  - 本頁無關聯入口按鈕；id / description / from 為可就地編輯欄，其餘欄唯讀。
  - loading 於 `core()` 內開啟，但在同步檢查（`isError` / rows 空）**之前**（`LayoutContentTargets.vue:834`）；填錯而早退時會先閃一下 loading——與全域 §5.1 canonical（`updateLoading` 應在同步檢查後）有出入，屬既有行為觀察，記錄供後續決定是否調整。
- **契約**：`saveTargets` 採 `core() + finally`，`finally` 一處關 loading（`LayoutContentTargets.vue:880`）；`updateTargets` 失敗由 `errTemp` 旗標短路後續、走失敗提示。
