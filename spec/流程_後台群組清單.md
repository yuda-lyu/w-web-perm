# 後台群組清單流程

## 觸發

管理員（瀏覽器使用者，經 w-ui-loginout 登入取得有效 token、`connState='csLogin'`）進入後台後，於 `LayoutContent.vue` 左側導覽點「權限群組」（`mmGrups`）進入群組清單頁（`LayoutContentGrups.vue`）。此頁以 ag-grid（`WAggridVue`）列出 `grups` 表，提供：編輯模式切換、顯示欄位挑選、新增 / 複製 / 刪除 / 儲存、`belongUsers` 群組成員入口按鈕、`cpemis` 群組擁有權限入口按鈕。

寫入端點 `updateGrups` 經 hapi `verifyConn` → `checkToken(token)` 守門（`server/WWebPerm.mjs:1155,1171`），**token 無效 / 過期一律拒絕連線**；是否限定管理員由部署方的 `verifyBrowserUser` 決定。

## 欄位邊界

| 欄位 | 前端處理 | 後端處理 |
|---|---|---|
| `id` | `funNew` 自動產生、清單預設隱藏（`kpHeadHide`） | `ltdtDiffByKey` 的 diff key 為 `name`（非 id） |
| `name` | `errItemsByName` 偵測空（`grupNameEmpty`）/ 重複（`grupNameDuplicate`），顯示警告 icon；編輯模式可就地編輯（`LayoutContentGrups.vue:676`） | `ckKey('name')` 再檢查空 / 重複，違反即 `Promise.reject`（`WWebPerm.mjs:406`） |
| `description` | 編輯模式可就地編輯（`LayoutContentGrups.vue:677`） | 隨 `grups` 列一併 insert / save |
| `from` | 編輯模式可就地編輯（`LayoutContentGrups.vue:678`，供外部批次變更使用），清單預設隱藏 | 隨列儲存 |
| `belongUsers` | 衍生顯示欄位：由群組被多少使用者使用計算文字（`grupBlngRnderNoUser` / `grupBlngRnderUsers`，`LayoutContentGrups.vue:820-831`）；顯示為按鈕，點擊開啟 `VeGrupBlngUsers` 對話框（`showVeGrupBlngUsers` 的 then / catch 皆空，**不回填**，`LayoutContentGrups.vue:1117-1122`） | 非 grups 表持久欄位，不隨列儲存 |
| `cpemis` | 顯示為按鈕，按鈕文字由 `getCpemisText` 依擁有權限數計算（`grupRnderCpemisNoPermission` / `grupRnderCpemisHasNPermissions`）；點擊開啟 `VeCpemis` 對話框；對話框 resolve 回 `cpemis` 字串後寫回該列、`refresh`、`isModified=true`（`LayoutContentGrups.vue:1162-1173`）；儲存為 JSON5 字串 | 隨 `grups` 列一併 insert / save |
| `userId` / `timeCreate` / `userIdUpdate` / `timeUpdate` | 新增 / 複製時填 `{待自動給予}` 佔位符（`grupAddIdNew`） | `procOrm` 於 insert / save 時自動補實值 |
| `order` | 由清單順序決定 | `updateTabItems` 依列順序重給（`WWebPerm.mjs:360`） |

## i18n 訊息粒度規則

| 觸發情境 | i18n 鍵（grep `server/procLang.mjs`） | 顯示位置 | 粒度規則 |
|---|---|---|---|
| name 空 | `grupNameEmpty` | cell 警告 icon + tooltip | **具體**：逐 cell 標示是哪一格出錯 |
| name 重複 | `grupNameDuplicate` | cell 警告 icon + tooltip | 具體（前端提示，後端 `ckKey` 亦擋） |
| Save 時有任一 name 錯誤 | `errInNames` | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | **彙整粗粒度**：name 各類錯誤聚合為 1 句擋存（前端攔截，不送後端） |
| rows 為空 | `grupAddEmpty` | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作前置檢查 |
| 儲存成功 | `grupSaveGrupsSuccess` | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作結果 |
| 儲存失敗（後端 reject，含 token 失效） | `grupSaveGrupsFail`（後接錯誤字串） | `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕） | 操作結果 |
| 開 belongUsers 對話框但群組 id / 資料取不到 | `grupEditBlngNoGrupId` / `grupEditBlngNoGrupData` | `$alert` toast | 防呆（正常流程不觸發） |
| belongUsers 欄按鈕文字 | `grupBlngRnderNoUser` / `grupBlngRnderUsers` | cell 按鈕內文字 | 衍生顯示（使用者數摘要） |
| cpemis 欄按鈕文字 | `grupRnderCpemisNoPermission` / `grupRnderCpemisHasNPermissions` | cell 按鈕內文字 | 衍生顯示（權限數摘要） |
| 非預期例外（外層 catch） | `anUnexpectedErrorOccurred` | `$alert` toast | 非預期錯誤低調呈現 |

> 粒度觀察（對齊全域 §6.2）：name 欄錯誤在 **cell 層為具體訊息（空 / 重複 2 種）**，但 Save 擋存時 `isError` 彙整為 **粗粒度 1 句**（`errInNames`）。本頁 `isError` 確實引用已定義的 `errItemsByName`（`LayoutContentGrups.vue:584-595`），故 name 空 / 重複由**前端攔截**並以 `$dg.showCheckYes` 持久 modal 呈現 `errInNames`、**不送後端**（異於 targets 之 `isError` 引用未定義變數恆空而走後端 `ckKey`）。寫測試時，欄位錯誤案例應斷言 cell tooltip 的**具體鍵**（`grupNameEmpty` / `grupNameDuplicate`），並斷言 Save 後 modal 出現前端彙整的 `errInNames`。

## spec 規則摘要

- **權限**：所有 `updateGrups` 寫入經 `verifyConn` → `checkToken` 守門（有效 token 才可寫）；是否限管理員由部署方 `verifyBrowserUser` 決定。
- **粒度**：name 欄錯誤走 cell 警告 icon + tooltip（具體 2 種：空 / 重複，僅前端視覺提示）；Save 擋存彙整與操作主要結果（前端 `errInNames` / rows 空 `grupAddEmpty` / 成功 `grupSaveGrupsSuccess` / 失敗 `grupSaveGrupsFail`）一律走 `$dg.showCheckYes` 持久 modal（System message 對話框 + OK 鈕，對齊全域 §5.1 禁原生 alert）；非預期例外與防呆（`grupEditBlngNoGrupId` / `grupEditBlngNoGrupData`）走 `$alert` toast 低調呈現。
- **邊界**：
  - `grups` **name 不可空 / 不可重複**（前端 `errItemsByName` 警告 + 後端 `ckKey('name')` 雙守門，`WWebPerm.mjs:405-410`）。
  - CRUD 為「單頁多動作 + 批次儲存」模型：`addItem` / `copyItem` / `deleteItemsCheck` / `showVeGrupBlngUsersById` / `showVeCpemisById` 只改 `opt.rows` 並設 `isModified=true`，**不即時打 API**；一律 Save 才以 `ltdtDiffByKey`（依 `name`）批次 del / insert / save 送出。
  - 稽核欄位（`userId` / `time*`）前端填 `{待自動給予}` 佔位符，後端 `procOrm` 補實值。
  - `belongUsers` 為衍生顯示欄位（由使用者 `cgrups` 反查群組成員數），非 grups 表持久欄位；其按鈕開 `VeGrupBlngUsers` 對話框後不回填本列資料。
  - `cpemis` 為群組所擁有權限清單（JSON5 字串），其按鈕開 `VeCpemis` 對話框後將 resolve 字串寫回該列並設 `isModified=true`，於 Save 隨列送出。
- **契約**：`saveGrups` 採 `core() + finally`，`finally` 一處關 loading（`LayoutContentGrups.vue:1239`）；`updateGrups` 失敗由 `errTemp` 旗標短路後續、走 `grupSaveGrupsFail` 持久 modal。
- **loading 時序**：loading 於 `core()` 內開啟，但在同步檢查（`isError` / rows 空）**之前**（`LayoutContentGrups.vue:1189`）；填錯而早退時會先閃一下 loading——與全域 §5.1 canonical（`updateLoading` 應在同步檢查後）有出入，屬既有行為觀察，記錄供後續決定是否調整。
